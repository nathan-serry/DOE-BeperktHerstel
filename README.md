# DOE - Beperkt Herstelbaarheid

## Project Overview
This project demonstrates limited recoverability (beperkte herstelbaarheid) concepts. The term refers to systems or data that have restricted ability to be restored or recovered after failures or data loss.

## Project Structure
```
DOE-BeperktHerstel/
├── index.html          # Main landing page
├── onderzoek.html      # Research/documentation page (Dutch)
├── style.css           # Styling for the web pages
├── .gitignore          # Git ignore rules
└── README.md           # This file
```

## Key Concepts
- **Limited Recoverability**: Systems designed with restricted restoration capabilities
- **Data Persistence**: Understanding what data can and cannot be recovered
- **System Resilience**: Building systems that acknowledge and work within recovery limitations

## Usage
1. Open `index.html` in a web browser to view the main page
2. Access `onderzoek.html` for detailed research and documentation
3. Styles are managed in `style.css`

## Development
This is a static web project. No build tools or dependencies are required.

## Language
The project primarily uses Dutch terminology, with "onderzoek" meaning "research" and "beperkte herstelbaarheid" meaning "limited recoverability".
